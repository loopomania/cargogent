2. Component Roles

Caddy

  • Role: Reverse proxy, SSL termination, routing.
  • Tech: Caddy v2.
  • Config: Caddyfile (prod) / Caddyfile.local (dev).
  • Routing: aigentinsure.com/api/* → backend:3000; aigentinsure.com/* → frontend:8080; n8n.aigentinsure.com → n8n:5678 (IP-restricted in prod)
  • Security: SSL, IP-based access control for n8n.

Frontend

  • Role: SPA UI.
  • Tech: React 19, Vite 7, Tailwind CSS, React Router.
  • Runtime: Nginx Alpine on port 8080.
  • Build: Multi-stage Docker build (Node build → Nginx serve).
  • Security: Non-root user (uid 101).

Backend

  • Role: REST API, auth, business logic, DB access.
  • Tech: Node.js 20, Express 4.
  • Runtime: Node Alpine, port 3000.
  • Features: JWT auth, OTP, bcrypt, CORS, rate limiting, Excel upload, data.gov.il sync.
  • Security: Read-only FS, no-new-privileges, non-root user.

n8n

  • Role: Workflow automation (OTP emails, sync triggers).
  • Tech: n8n (Node.js).
  • Runtime: n8nio/n8n image, port 5678.
  • Storage: Neon PostgreSQL (prod) or SQLite (dev).
  • Workflows: OTP email, Programs and Plans sync.

Database

  • Role: Persistent storage.
  • Tech: PostgreSQL 15.
  • Deployment: Neon (prod) or local Postgres container (dev).
  • Schema: users, clients, providers, program_performances, n8n schema.

4. Data Flow

  • Auth: Frontend → Backend → DB (users, OTP). Backend → n8n webhook for OTP emails.
  • API: Frontend → Caddy → Backend → DB.
  • Sync: n8n or Settings UI → Backend → data.gov.il → DB.
  • Static: Caddy → Frontend (Nginx) → static files.

5. Strengths

  • Separation of concerns: Frontend, API, automation, DB.
  • Security: Non-root, read-only FS, CORS, rate limiting, IP restrictions.
  • Deployment: Docker Compose, dev/prod compose files.
  • DB choice: Managed Neon for prod.
  • Flexibility: n8n for workflows without code changes.

6. Considerations

  • Caddyfile webhook path: ARCHITECTURE.md mentions http://n8n:5678/webhook/otpme; the actual OTP webhook is register-otp.
  • CORS: Backend allows https://aigentinsure.com and http://localhost:5173; prod domain is aigentinsure.com.
  • n8n DB: Prod uses Neon; dev uses SQLite for n8n to avoid PostgreSQL SSL issues.
  • Frontend nginx: /api proxy in nginx.conf is not used when Caddy routes /api/* directly to the backend.
